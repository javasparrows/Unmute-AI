# AI パイプライン設計

## 翻訳パイプライン

1. ユーザー入力 → 800msデバウンス → API呼び出し
2. Gemini API にシステムプロンプト + ユーザーテキストを送信
3. ストリーミングレスポンスをリアルタイムで反対パネルに反映
4. abort制御: 新しい入力があれば前のリクエストをキャンセル

## プロンプト設計

### 翻訳プロンプト

- 段落1:1対応を厳密に強制（N段落入力 → N段落出力）
- 段落の結合・分割を禁止
- 説明やコメントの付加を禁止
- ジャーナルごとのスタイルガイドをプロンプトに含める
- 改行、段落、スペースなどのフォーマットをそのまま保持

### 言語検出プロンプト

- テキスト先頭200文字をサンプルとして送信
- ISO 639-1コードのみを返すよう指示

### 論理構成チェックプロンプト

- 段落ごとの役割（Introduction, Results等）を分析
- 論理フロー（前後段落との接続）を評価
- 改善提案（最大3件）とスコア（1-10）を返す
- JSON形式で出力を強制

## ループ防止

- `translationSource` ref フラグで翻訳元パネルを追跡
- 左パネル編集 → 右パネル更新時は右パネルのonChangeを無視
- 右パネル編集 → 左パネル更新時は左パネルのonChangeを無視
