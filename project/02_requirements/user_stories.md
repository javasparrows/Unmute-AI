# ユーザーストーリー

## US-1: 日本語から英語への翻訳

研究者として、左パネルに日本語を入力し、同期ボタン（→）を押したら右パネルに英語翻訳が表示されてほしい。変更した文だけが翻訳されるので、未変更部分の翻訳が崩れず安心して編集したい。

## US-2: 英語の修正と逆翻訳

研究者として、右パネルの英語を修正し、同期ボタン（←）を押したら左パネルの日本語も対応して更新されてほしい。英語側の修正内容が日本語にどう反映されるか確認したい。

## US-3: ペーストによる言語自動検出

研究者として、テキストをペーストしたら自動で言語が検出され、言語設定が更新されてほしい。その後、同期ボタンを押して翻訳を実行したい。

## US-4: カーソル位置の確認（アラインメントベース）

研究者として、一方のパネルで文をクリックしたら、もう一方のパネルで意味的に対応する文がハイライトされてほしい。翻訳で複数の文が1つに統合されていても、対応する全ての文がハイライトされてほしい。

## US-5: ジャーナルに合わせた翻訳

研究者として、投稿先ジャーナル（Nature, Science等）を設定したら、そのジャーナルのスタイルに合った翻訳が出力されてほしい。

## US-6: 論文構成の確認

研究者として、書いた文章の段落ごとの論理構成をAIにチェックしてもらい、改善点を確認したい。

## US-7: 修正履歴の復元

研究者として、過去の翻訳内容を履歴から復元できるようにしてほしい。作業の途中経過を失いたくない。

## US-8: DB保存

**As a** 研究者
**I want to** 翻訳の現在の状態を「保存」ボタンでDBに保存したい
**So that** ブラウザを閉じたり端末を変えてもデータが失われない

## US-9: バージョン復元

**As a** 研究者
**I want to** 過去のバージョンを選択して復元したい
**So that** 翻訳の過程を振り返ったり、以前の状態に戻せる

## US-10: 別端末からのアクセス

**As a** 研究者
**I want to** 別のブラウザやPCからログインしても同じドキュメントにアクセスしたい
**So that** 場所を問わず翻訳作業を続けられる

## US-11: ログイン

**As a** 研究者
**I want to** Google アカウントでログインしたい
**So that** 自分専用のドキュメント管理スペースを持てる

## US-12: 複数ドキュメント管理

**As a** 研究者
**I want to** 複数の論文翻訳を別々のドキュメントとして管理したい
**So that** プロジェクトごとに翻訳を整理できる
