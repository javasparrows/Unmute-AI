# 機能要件

## FR-1: 双方向手動同期翻訳

- 左側に元の文章、右側に翻訳後の文章を表示する
- 両側を自由に編集できる（編集時に自動翻訳は発生しない）
- パネル間に配置された同期ボタン（→ / ←）を押すことで翻訳を実行する
  - 「→」ボタン: 左パネルの変更を翻訳して右パネルに反映
  - 「←」ボタン: 右パネルの変更を翻訳して左パネルに反映
- 前回同期時のスナップショットと比較し、変更があった文のみを翻訳する（差分翻訳）
- 翻訳エンジンは Gemini 2.5 Flash を使用（DeepLは廃止）
- 文が多い場合は最大4並列でAPI呼び出しし、レイテンシを低減する
- LaTeXコマンド（`\cite{}`, `\ref{}`, `\label{}` 等）は翻訳時に保持し、翻訳先の適切な位置に配置する

## FR-2: 言語設定・自動検出

- 左側・右側それぞれの言語を設定できる
- 文章をペーストしたら自動で言語検出し、言語セレクターを更新する
- ペースト後の翻訳は同期ボタン押下時に実行される（自動翻訳は行わない）

## FR-3: フォーマット保持

- 改行、段落、スペースなどはそのまま反映されるようにプロンプトを設計する
- 日本語への翻訳時は「だ・である調」（常体）を使用する（学術論文の標準的な文体）

## FR-4: カーソル連動（アラインメントベース）

- カーソルの位置が左側と右側で連動する
- 翻訳後にLLMアラインメントAPIでソース↔ターゲットのN:M対応マップを取得する
- 一方のパネルで文をクリックしたら、アラインメントマップで対応する文（1つ以上）がもう一方のパネルでハイライトされる
- 翻訳で文が統合・分割されても正しく対応する（N:M対応）
- アラインメント未取得時は位置ベースの1:1マッピングにフォールバック

## FR-5: 修正履歴

- 修正履歴が残る

## FR-6: 投稿先ジャーナル設定

- 想定の投稿先ジャーナルを設定できる
- ジャーナル設定によって翻訳プロンプトが変わる

## FR-7: 論理構成チェック

- 段落ごとの論理構成をチェックできる機能
- Gemini APIに投げて分析結果を表示する

## FR-8: DB永続化 (Database Persistence)

- ドキュメントデータはPostgreSQL (Cloud SQL) に永続化される
- Prisma ORMを使用してタイプセーフなDB操作を実現
- ページロード時にDBから最新バージョンを自動読み込み
- sentence rangesもバージョンに含めて保存し、リロード時のマッピング精度を保証

## FR-9: バージョン管理 (Version Management)

- 「保存」ボタンで現在の状態を新しいバージョンとしてDBに保存
- 保存項目: 原文, 翻訳, 言語, journal, provider, leftRanges, rightRanges
- バージョン履歴パネルで過去バージョンの一覧表示
- 任意のバージョンをクリックしてその時点の状態に復元
- バージョン番号は自動インクリメント (v1, v2, v3, ...)

## FR-10: ユーザー認証 (User Authentication)

- Google OAuth 2.0によるユーザー認証 (Auth.js v5)
- 未認証ユーザーは /login ページにリダイレクト
- ユーザーごとにドキュメントを分離
- セッション管理はAuth.js + Prisma Adapterによる永続セッション

## FR-11: 複数ドキュメント管理 (Multi-Document Management)

- ダッシュボード (/) でドキュメント一覧を表示
- 新規ドキュメント作成、削除、名前変更
- 各ドキュメントにバージョン履歴を持つ
- ドキュメント間の切り替えはURL遷移 (/documents/[id])

## FR-12: API使用量詳細ログ (API Usage Logging)

- API呼び出しごとに詳細なログを `ApiUsageLog` テーブルに記録する
- 記録項目: userId, type, model, inputTokens, outputTokens, sourceChars, translatedChars, sourceLang, targetLang, documentId, createdAt
- 対象エンドポイント:
  - `POST /api/translate-sentence` → type="translation"
  - `POST /api/check-structure` → type="structure_check"
- 既存の `UsageRecord`（月次プラン制限チェック用）とは独立して運用する
- SaaS運用のコスト分析・ユーザー別使用量追跡に活用する

## FR-13: CI/CD自動デプロイ (Continuous Deployment)

- mainブランチへのpushで自動的にCloud Runへデプロイされる
- デプロイ前にPrismaマイグレーションが自動実行される（失敗時はデプロイ中止）
- PRに対してlint + buildの自動チェックが実行される
