# バリデーション計画

## 前提

- 翻訳は手動同期（→/←ボタン押下）で実行される（自動翻訳は発生しない）
- ハイライト同期はアラインメントベース（翻訳後にLLMが文対応を分析）
- テストデータは必ず `test_cases.md` のアカデミックテキストを使用する

## 検証手順

### Step 1: 準備
1. `yarn dev` で開発サーバーを起動
2. Chrome で http://localhost:3000/ を開く
3. DevTools Console を開いておく（エラー監視用）

### Step 2: 基本翻訳フロー（TC-07 自動翻訳が発生しないこと）
1. 左パネルに `test_cases.md` のアカデミックテキストをペースト
2. **確認**: 右パネルに自動で翻訳が表示されないこと
3. 「→」ボタンを押す
4. **確認**: 翻訳が右パネルに表示されること

### Step 3: 文数の数値検証（最重要 — 漏れやすい）
翻訳が表示されたら、**必ず DevTools Console で以下を実行**:
```javascript
const editors = document.querySelectorAll('.tiptap');
const leftText = editors[0]?.textContent || '';
const rightText = editors[1]?.textContent || '';
const REGEX = /([.。!?！？])/g;
function split(text) {
  return text.split('\n\n').flatMap((p, i) => {
    const r = []; let last = 0; REGEX.lastIndex = 0; let m;
    while ((m = REGEX.exec(p)) !== null) {
      r.push(p.slice(last, m.index + 1)); last = m.index + 1;
    }
    if (last < p.length) r.push(p.slice(last));
    return r.filter(s => s.trim());
  });
}
const L = split(leftText), R = split(rightText);
console.log(`Left: ${L.length}文, Right: ${R.length}文, 差: ${L.length - R.length}`);
if (L.length !== R.length) console.warn('⚠️ 文数不一致 → アラインメントが必須');
```
- 文数の差を記録する（アラインメントが正しく動作していれば、差があっても問題ない）

### Step 4: TC-02 翻訳→原文ハイライト（最もズレやすい方向を先にテスト）
1. **右パネル**で以下の文をクリックし、左パネルのハイライトを確認:
   - 1段落目の1文目
   - `e.g.` や `et al.` を含む文の直後の文 ← **偽文境界の影響を受ける箇所**
   - 1段落目の最終文（COVID-19...）
   - 2段落目の1文目
   - 2段落目の最終文
2. **判定基準**: すべての文で原文のハイライトが意味的に対応する文を指していること
3. N:M対応（統合・分割）がある場合、複数文が同時ハイライトされること

### Step 5: TC-01 原文→翻訳ハイライト
1. **左パネル**で各文をクリック
2. 翻訳パネルの対応文がハイライトされることを確認
3. `\cite{...}` やダッシュ `---` を含む文でも正確に対応すること

### Step 6: TC-08 差分翻訳
1. 左パネルの1文だけを変更する
2. 「→」ボタンを押す
3. **確認**: 変更した1文のみが翻訳更新され、他の文は前回の翻訳がそのまま保持

### Step 7: TC-09 逆方向同期
1. 右パネルの翻訳を手動で修正する
2. 「←」ボタンを押す
3. **確認**: 左パネルの対応する文が更新されること

### Step 8: TC-03 リロード永続化
1. テキストが入った状態でページをリロード（Cmd+R）
2. **確認**: 原文・翻訳文が両方とも復元されている
3. 言語設定・ジャーナル設定・プロバイダ設定も保持されていること

### Step 9: TC-04 クリア後の再入力
1. 「クリア」ボタンを押す
2. 新しいテキスト（文の数が異なるもの）を入力し「→」ボタンで同期する
3. **確認**: エラーなく翻訳が実行される

### Step 10: TC-14 並列翻訳の順序保持
1. 20文以上の長いテキストを原文パネルにペーストし「→」ボタンで同期
2. **確認**: 翻訳結果の文順が原文の文順と正しく対応していること
3. DevToolsのNetworkタブで並列リクエスト（最大4本）が確認できること

### Step 11: TC-13 アラインメントフォールバック
1. アラインメントAPIがエラーを返す、または取得完了前にクリックする
2. **確認**: 位置ベースの1:1マッピングでフォールバック動作すること
3. エラーが発生してもアプリがクラッシュしないこと

## 合否判定

| 結果 | 判定 |
|---|---|
| Step 3 の文数検証を実施 + Step 4〜11 すべて pass | リリース可 |
| Step 3 を省略した | **不合格**（文数検証は必須） |
| Step 4 (右→左ハイライト) が fail | アラインメントの修正が必要 |
| Step 6 (差分翻訳) が fail | use-sync-translation の修正が必要 |
| Step 10 (並列順序) が fail | チャンク分割ロジックの修正が必要 |

## 注意事項

- **Step 4（右→左方向）を必ず Step 5（左→右方向）より先にテストする** — ズレが出やすい方向を先に検証する
- **Step 3（文数検証）は省略不可** — 過去にこの検証を省略して mapping バグが漏れた
- build pass のみで完了としない — ハイライト同期は TypeScript の型チェックでは検出できない
- 短い3文テストだけで pass にしない — 必ず `test_cases.md` のアカデミックテキストを使う
