# バリデーション計画

## 検証手順

### Step 1: 準備
1. `yarn dev` で開発サーバーを起動
2. Chrome で http://localhost:3000/ を開く
3. localStorage をクリア（DevTools → Application → Local Storage → Clear）

### Step 2: TC-04 クリア後再入力（クラッシュ修正確認）
1. 短いテキスト（3文）を原文パネルに入力、翻訳完了を待つ
2. 「クリア」押下
3. 長いテキスト（test_cases.md のアカデミックテキスト）をペースト
4. **確認**: コンソールにエラーなし、翻訳が表示される

### Step 3: TC-01 原文→翻訳ハイライト
1. 原文パネルで 1 段落目の各文をクリック
2. 翻訳パネルの対応文がハイライトされることを確認
3. 2 段落目でも同様に確認

### Step 4: TC-02 翻訳→原文ハイライト（メインバグ）
1. 翻訳パネルで以下の文をクリックし、原文パネルのハイライトを確認:
   - 1段落目の1文目（EHRs contain...）
   - 1段落目の5文目（COVID-19 pandemic...）← **以前ズレていた箇所**
   - 1段落目の6文目（最終文）
   - 2段落目の1文目（Clinical information extraction...）
   - 2段落目の最終文
2. **判定基準**: すべての文で原文・翻訳のハイライトが同じ意味の文を指していること

### Step 5: TC-03 リロード永続化
1. テキストが入った状態でページをリロード（Cmd+R）
2. **確認**: 原文・翻訳文が復元されている
3. 言語設定（左: English, 右: 日本語）が保持されている
4. もう一度リロードしても同じ

### Step 6: TC-05 略語テスト
1. 翻訳パネルで `\cite{...}` を含む文の前後をクリック
2. **確認**: ハイライトが隣の文にズレていない

## 合否判定
- TC-01〜TC-06 すべて pass → リリース可
- TC-02 が fail → ハイライト同期の根本修正が必要
- TC-04 が fail → `detectChangedSentences` の修正確認
