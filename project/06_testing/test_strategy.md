# テスト戦略

## テスト観点

### 1. ハイライト同期（双方向）
- 改行、段落が翻訳前と翻訳後で一致する
- 選択範囲のハイライトが翻訳前と翻訳後で一致する（**双方向**）
  - 原文クリック → 翻訳のハイライトが正確
  - 翻訳クリック → 原文のハイライトが正確（**以前のバグ: 1つズレ**）
- ハイライトが文字の高さと合っている。ずれていない
- 段落をまたいだ場合でもハイライト index がズレない

### 2. 文分割の一貫性
- 原文パネルと翻訳パネルの文 index 算出に**同一の `splitSentences` トークンベースロジック**を使用
- 以前の問題: 原文パネルは regex (`/[.。!?！？]\s*/g`)、翻訳パネルはトークンベースで、略語等のピリオドで文数が不一致 → index ズレ
- 修正後: 両パネルとも `computeSentenceRanges(splitSentences(text))` を使用

### 3. テキスト入力・永続化
- 文字、文章の paste ができる
- テキスト入力後に debounce (800ms) で自動翻訳が発動する
- **リロード後もテキストが保持される**（localStorage: `editor-left-text`, `editor-right-text`）
- 言語設定・ジャーナル設定・プロバイダ設定もリロード後に保持される

### 4. エラー耐性
- クリア後に文数が異なるテキストを入力しても `TypeError` が発生しない
- `detectChangedSentences` が現在のテキストの文数を超えるインデックスを返さない
- API エラー時にエラーメッセージが表示され、クラッシュしない

## テストデータ
- **必ずアカデミックテキスト**（`\cite{...}`, `---`, 略語含む）でテストする
- 短い 3 文テストだけで pass にしない（略語やピリオドの多い長文で再現するバグがある）
- `project/06_testing/test_cases.md` のテストテキストを使用する
