GitHub Actions が Cloud Run にデプロイする際、Secret Manager の情報を自動的に設定します。

他のプロジェクトで真似する方法

このパターンは以下の順序で実装できます：

ステップ1: ローカル環境

1. compose.yml を作成してデータベースを定義
2. start_app.sh で docker compose up -d を実行
3. 接続文字列のデフォルト値を localhost に設定

ステップ2: Terraform でインフラ構築

1. database.tf で Cloud SQL インスタンスを定義
2. secret_manager.tf で接続文字列を保存
3. cloudrun.tf で env_secret_vars を設定

ステップ3: アプリケーション側

1. 環境変数から接続文字列を読み取る
2. デフォルト値として localhost の接続文字列を設定

ステップ4: CI/CD

1. GitHub Actions で Cloud Run にデプロイ
2. Secret Manager の参照を設定

この設計の利点：
- 環境分離: ローカルと本番で同じコードが動く
- セキュリティ: 本番の接続情報はSecret Managerで管理
- 自動化: Terraformがインフラを自動作成
- シンプル: 環境変数1つで切り替え可能
